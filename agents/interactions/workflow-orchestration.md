# 工作流编排交互逻辑

## 概述

定义多步骤任务的编排模式，协调多个能力模块完成复杂工作流。

## 编排原则

1. **步骤原子化**: 每个步骤独立可验证，失败可回滚
2. **依赖显式化**: 步骤间的依赖关系必须明确声明
3. **并行优先**: 无依赖关系的步骤并行执行
4. **渐进交付**: 优先完成核心路径，逐步补充完善

## 标准工作流

### WF-001: 新功能开发

```
需求分析 → 数据模型设计 → API 开发 → 前端开发 → 集成测试 → 代码审查
```

| 步骤 | 能力模块 | 输入 | 输出 | 验证 |
|------|----------|------|------|------|
| 需求分析 | user-communication | 用户需求 | 需求文档 | 用户确认 |
| 模型设计 | database-operations | 需求文档 | 模型定义 | Ruff 通过 |
| API 开发 | api-development | 模型定义 | 路由+Schema | pytest 通过 |
| 前端开发 | frontend-development | API 定义 | 组件+页面 | ESLint+Build 通过 |
| 集成测试 | testing | 全部代码 | 测试用例 | 测试通过 |
| 代码审查 | code-review | 全部变更 | 审查报告 | 无严重问题 |

**并行机会**: API 开发和前端组件骨架可并行进行

### WF-002: Bug 修复

```
问题复现 → 根因定位 → 实施修复 → 回归测试 → 审查确认
```

| 步骤 | 能力模块 | 输入 | 输出 | 验证 |
|------|----------|------|------|------|
| 问题复现 | testing | Bug 描述 | 复现步骤 | 确认可复现 |
| 根因定位 | error-handling | 复现步骤 | 根因分析 | 逻辑验证 |
| 实施修复 | api/fe/db | 根因分析 | 代码修复 | Lint 通过 |
| 回归测试 | testing | 修复代码 | 测试结果 | 全部通过 |
| 审查确认 | code-review | 修复变更 | 审查报告 | 无新问题 |

### WF-003: 部署发布

```
代码检查 → 测试执行 → 镜像构建 → 服务部署 → 健康检查
```

| 步骤 | 能力模块 | 输入 | 输出 | 验证 |
|------|----------|------|------|------|
| 代码检查 | code-review | 代码仓库 | 检查报告 | 无严重问题 |
| 测试执行 | testing | 代码仓库 | 测试报告 | 全部通过 |
| 镜像构建 | deployment | 代码仓库 | Docker 镜像 | 构建成功 |
| 服务部署 | deployment | Docker 镜像 | 运行容器 | 容器正常 |
| 健康检查 | deployment | 运行服务 | 健康状态 | 服务可用 |

### WF-004: 安全加固

```
安全扫描 → 风险评估 → 加固实施 → 验证测试 → 审计报告
```

| 步骤 | 能力模块 | 输入 | 输出 | 验证 |
|------|----------|------|------|------|
| 安全扫描 | security-audit | 代码仓库 | 漏洞列表 | 扫描完成 |
| 风险评估 | security-audit | 漏洞列表 | 优先级排序 | 评估完成 |
| 加固实施 | api/fe/db | 修复方案 | 代码修复 | Lint 通过 |
| 验证测试 | testing | 修复代码 | 测试结果 | 全部通过 |
| 审计报告 | security-audit | 全部结果 | 最终报告 | 报告完成 |

## 工作流选择指南

| 用户意图 | 推荐工作流 |
|----------|-----------|
| "新增一个XX功能" | WF-001 |
| "修复一个Bug" | WF-002 |
| "部署到生产环境" | WF-003 |
| "检查安全问题" | WF-004 |
| "优化代码" | WF-001 (部分步骤) |
| "添加测试" | testing 能力模块直接调用 |

## 异常处理

工作流执行中的异常处理策略：

| 异常类型 | 处理方式 |
|----------|----------|
| 步骤验证失败 | 停止后续步骤，报告失败原因 |
| 依赖步骤失败 | 跳过依赖此步骤的后续步骤 |
| 部分并行步骤失败 | 等待其他并行步骤完成，汇总结果 |
| 用户中断 | 保存当前进度，提供恢复点 |
