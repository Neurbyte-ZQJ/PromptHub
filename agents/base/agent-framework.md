# 智能体基础框架

## 概述

PromptHub 智能体基础框架定义了所有智能体的核心协议、生命周期和交互规范。每个智能体都遵循统一的基础结构，确保模块化开发与无缝集成。

## 智能体生命周期

```
初始化 → 加载配置 → 注册能力 → 接收任务 → 执行处理 → 返回结果 → 清理资源
```

### 阶段说明

| 阶段 | 职责 | 输入 | 输出 |
|------|------|------|------|
| 初始化 | 读取智能体定义，验证完整性 | 智能体配置 | 初始化状态 |
| 加载配置 | 合并全局配置与智能体配置 | 配置文件 | 运行时配置 |
| 注册能力 | 加载所需能力模块 | 能力列表 | 可用能力集合 |
| 接收任务 | 解析用户意图，匹配能力 | 用户输入 | 任务描述 |
| 执行处理 | 调用能力模块执行任务 | 任务描述 + 上下文 | 执行结果 |
| 返回结果 | 格式化输出，附加元信息 | 执行结果 | 结构化响应 |
| 清理资源 | 释放临时状态 | 无 | 清理完成 |

## 智能体接口规范

### 标准属性

每个智能体必须定义以下属性：

```json
{
  "id": "string - 唯一标识符",
  "name": "string - 显示名称",
  "description": "string - 功能描述",
  "version": "string - 语义化版本号",
  "capabilities": ["string - 引用的能力模块 ID"],
  "triggers": ["string - 触发关键词"],
  "scope": "string - 作用域 (backend/frontend/fullstack/infra)",
  "priority": "number - 优先级 (1-10, 1最高)"
}
```

### 标准方法

| 方法 | 说明 | 返回 |
|------|------|------|
| `can_handle(task)` | 判断是否能处理指定任务 | boolean |
| `execute(task, context)` | 执行任务 | Result |
| `validate(result)` | 验证执行结果 | ValidationResult |
| `rollback(task)` | 任务失败时回滚 | void |

## 智能体分类

| 类别 | 作用域 | 说明 |
|------|--------|------|
| `backend` | 后端 | API 开发、数据库操作、认证逻辑 |
| `frontend` | 前端 | 组件开发、页面构建、样式处理 |
| `fullstack` | 全栈 | 跨前后端的功能开发 |
| `infra` | 基础设施 | 部署、CI/CD、容器化 |
| `quality` | 质量保障 | 测试、审查、安全审计 |

## 与现有系统集成

智能体框架与项目现有结构的关系：

- **`.claude/commands/`**: 命令是智能体的触发入口，命令调用智能体执行
- **`.trae/skill-config.json`**: 技能配置与智能体能力模块对应
- **`.trae/rules/project_rules.md`**: 项目规则作为智能体的全局约束
- **`CLAUDE.md`**: 项目概览作为智能体的上下文知识
