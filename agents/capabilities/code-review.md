# 代码审查能力模块

## 概述

提供代码质量审查、安全审计和最佳实践检查的能力。

## 能力标识

- **ID**: `capability.code-review`
- **版本**: 1.0.0
- **作用域**: fullstack
- **触发关键词**: 代码审查, 审查代码, review, 代码质量, PR审查

## 能力清单

### CAP-CR-001: 代码质量审查

检查代码的结构质量和可维护性。

**审查项**:
- 函数/方法职责单一性
- 命名规范一致性（snake_case / PascalCase）
- 代码重复和抽象合理性
- 错误处理完善性
- 类型注解完整性

### CAP-CR-002: 安全审计

检查代码的安全漏洞和风险。

**审查项**:
- 认证和授权逻辑正确性
- 数据可见性控制（用户自己的 + is_public=True）
- SQL 注入防护（ORM 参数化查询）
- 敏感信息泄露（密码哈希、JWT 密钥）
- CORS 配置合理性
- 输入验证完整性

### CAP-CR-003: 后端规范审查

检查后端代码是否符合项目规范。

**审查项**:
- 路由定义在 `backend/main.py` 中
- Pydantic v2 语法正确性
- SQLAlchemy 2.0 风格一致性
- 分页响应头正确返回
- Import 排序符合 isort 规范
- 错误消息使用中文

### CAP-CR-004: 前端规范审查

检查前端代码是否符合项目规范。

**审查项**:
- 函数式组件 + Hooks
- Tailwind CSS + cn() 使用
- 路径别名 @ 正确使用
- API 调用在 api.ts 中定义
- 认证 API 通过 useApi() 调用
- 弹窗状态由 Layout 管理

### CAP-CR-005: 性能审查

检查代码的性能问题。

**审查项**:
- 数据库查询无 N+1 问题
- 合理使用 eager loading（`selectin` / `joined`）
- 前端组件 memo/useMemo/useCallback 使用
- 分页查询使用数据库级别分页
- 避免不必要的数据加载

## 审查输出格式

```
## 审查报告

### 摘要
- 审查文件数: N
- 严重问题: N
- 建议改进: N
- 亮点: N

### 严重问题 (必须修复)
- [文件:行号] 问题描述 → 修复建议

### 建议改进 (推荐修复)
- [文件:行号] 问题描述 → 改进建议

### 亮点 (值得保持)
- 良好实践描述
```
