# 安全审计能力模块

## 概述

提供代码安全扫描、漏洞检测和安全最佳实践检查的能力。

## 能力标识

- **ID**: `capability.security-audit`
- **版本**: 1.0.0
- **作用域**: fullstack
- **触发关键词**: 安全审计, 安全检查, 漏洞, 安全风险, XSS, SQL注入, CSRF

## 能力清单

### CAP-SEC-001: 认证安全

检查认证和授权机制的安全性。

**审查项**:
- JWT 密钥强度和轮换策略
- Token 有效期合理性（当前 60 分钟）
- 密码哈希算法安全性（bcrypt）
- Token 存储安全性（localStorage vs HttpOnly Cookie）
- 刷新 Token 机制
- 密码重置流程安全性

### CAP-SEC-002: 数据安全

检查数据访问和存储的安全性。

**审查项**:
- 数据可见性控制正确性
- 私有数据访问控制
- 敏感数据加密存储
- 数据库连接安全
- 备份和恢复策略

### CAP-SEC-003: 输入验证

检查用户输入的验证和清洗。

**审查项**:
- API 输入参数验证（Pydantic Schema）
- SQL 注入防护（ORM 参数化查询）
- XSS 防护（React 自动转义 + 内容安全策略）
- 文件上传安全（类型/大小限制）
- 请求体大小限制

### CAP-SEC-004: 网络安全

检查网络通信和配置的安全性。

**审查项**:
- CORS 配置合理性（当前允许所有来源）
- HTTPS 强制（生产环境）
- API 速率限制
- 请求头安全配置
- Cookie 安全属性

### CAP-SEC-005: 依赖安全

检查第三方依赖的安全漏洞。

**审查项**:
- Python 依赖已知漏洞
- Node.js 依赖已知漏洞
- 依赖版本是否过时
- 锁文件完整性

**检查命令**:
```bash
pip audit                    # Python 依赖审计
cd frontend && npm audit     # Node.js 依赖审计
```

## 安全等级定义

| 等级 | 说明 | 处理要求 |
|------|------|----------|
| CRITICAL | 可被直接利用的漏洞 | 立即修复 |
| HIGH | 严重安全风险 | 优先修复 |
| MEDIUM | 潜在安全问题 | 计划修复 |
| LOW | 安全加固建议 | 建议修复 |

## 审计输出格式

```
## 安全审计报告

### 摘要
- CRITICAL: N
- HIGH: N
- MEDIUM: N
- LOW: N

### 严重漏洞 (CRITICAL)
- [文件:行号] 漏洞描述 → 风险说明 → 修复方案

### 高风险问题 (HIGH)
- [文件:行号] 问题描述 → 风险说明 → 修复方案

### 中等风险 (MEDIUM)
- [文件:行号] 问题描述 → 建议方案

### 低风险建议 (LOW)
- 建议描述
```
